next-forge provides a flexible CMS package that you can integrate with any content management system of your choice.

## Getting Started

The CMS package has been designed to work with any CMS solution. You can choose from:

- **Headless CMS**: Strapi, Sanity, Contentful, Hygraph, etc.
- **File-based**: MDX, Markdown with frontmatter
- **Database-driven**: Custom solution with your existing database
- **Git-based**: Content stored directly in your repository

### Implementation Guide

1. **Choose your CMS solution** based on your needs and preferences
2. **Implement the standardized interfaces** provided in the package
3. **Configure environment variables** for your chosen CMS
4. **Customize components** to match your content structure
5. **Test your implementation** with the provided API structure

The package provides TypeScript interfaces for blog posts and legal pages that you can implement with any CMS backend.

### API Structure

The CMS package exports a consistent API regardless of your chosen backend:

```typescript
// Blog API
export const blog = {
  getPosts: async (): Promise<PostMeta[]> => { /* Your implementation */ },
  getLatestPost: async (): Promise<Post | null> => { /* Your implementation */ },
  getPost: async (slug: string): Promise<Post | null> => { /* Your implementation */ },
}

// Legal Pages API
export const legal = {
  getPosts: async (): Promise<LegalPost[]> => { /* Your implementation */ },
  getLatestPost: async (): Promise<LegalPost | null> => { /* Your implementation */ },
  getPost: async (slug: string): Promise<LegalPost | null> => { /* Your implementation */ },
}
```

### Components

The package includes placeholder components that you can customize for your CMS:

- **Body**: Rich text content rendering
- **CodeBlock**: Syntax-highlighted code blocks
- **Feed**: Content listing and pagination
- **Image**: Optimized image handling
- **TableOfContents**: Document navigation
- **Toolbar**: CMS editing interface (if needed)

## Migration Benefits

By using a custom CMS implementation, you get:

- **Full Control**: Complete ownership of your content and data
- **Cost Efficiency**: No external service fees
- **Flexibility**: Choose the exact features you need
- **Performance**: Direct integration with your infrastructure
- **Privacy**: Content stays within your control

For detailed implementation instructions, see the [CMS Package README](https://github.com/haydenbleasel/next-forge/tree/main/packages/cms#readme).
